# 支付宝公益捐款自动点击器 - 安装与使用说明

> ⚠️ 此工具涉及真实金钱交易，请务必先用最小金额（0.01元）测试，确认流程正常后再正式使用。

---

## 一、环境要求

| 软件 | 版本要求 | 下载地址 |
|------|----------|----------|
| Node.js | v18 或以上（推荐 v20+） | https://nodejs.org |
| npm | 随 Node.js 一起安装 | — |

安装完成后，打开终端（PowerShell 或 CMD）验证：

```powershell
node --version   # 应显示 v18.x.x 或更高
npm --version    # 应显示 9.x.x 或更高
```

---

## 二、安装步骤

### 第一步：安装项目依赖

在项目根目录下执行：

```powershell
npm install
```

### 第二步：安装 Playwright 浏览器

```powershell
npx playwright install chromium
```

> 此命令会下载 Chromium 浏览器内核，约 100–200 MB，需要等待几分钟。

---

## 三、配置文件说明

项目有两个需要配置的文件：

### 1. `.env` 文件（敏感信息）

复制示例文件：

```powershell
copy .env.example .env
```

用文本编辑器打开 `.env`，按以下说明填写：

```env
# 支付宝支付密码（6位数字）
PAY_PASSWORD=123456

# 目标银行卡名称（付款时使用哪张卡）
# 填写银行全称，如：中国农业银行、招商银行、中国工商银行
TARGET_BANK_CARD_NAME=中国农业银行

# 银行卡尾号后四位（如果同一银行有多张卡，填此项以区分）
TARGET_BANK_CARD_LAST_FOUR=6715

# 浏览器用户数据目录（用于保存登录状态，无需修改）
USER_DATA_DIR=./data/browser-user-data
```

> ⚠️ `.env` 文件包含你的支付密码，绝对不要分享给他人或上传到网络。

### 2. `config/default.json` 文件（捐款参数）

```json
{
  "donation": {
    "url": "https://love.alipay.com/donate/itemDetail.htm?name=2016051810442526101",
    "amount": 0.01,
    "totalTimes": 3,
    "minDelay": 5000,
    "maxDelay": 5000
  }
}
```

| 字段 | 说明 | 示例 |
|------|------|------|
| `url` | 捐款项目页面地址，不同公益项目地址不同 | — |
| `amount` | 每次捐款金额（元） | `0.01` |
| `totalTimes` | 总共捐款次数 | `10` |
| `minDelay` | 两次捐款之间的等待时间（毫秒），建议不低于 5000 | `5000` |
| `maxDelay` | 同上（目前与 minDelay 保持一致即为固定等待） | `5000` |

## 四、首次使用：登录支付宝

**每次更换账号或登录状态失效时，都需要重新执行此步骤。**

```powershell
npm run login
```

执行后会自动打开浏览器，进入支付宝登录页面：

1. 在浏览器中手动完成登录（账号密码登录或扫码登录均可）
2. 看到支付宝首页，确认登录成功
3. 回到终端，**按 Enter 键**
4. 程序自动保存登录状态（cookies），浏览器关闭

```
请在浏览器中完成登录操作...
登录成功后，回到此终端按 Enter 键保存并退出
[按 Enter]
Cookies 已保存到: data/cookies.json
共保存 XX 个 cookie
浏览器已关闭，登录状态已保存。
```

> 登录状态通常可以保持数天至数周，失效后重新执行 `npm run login` 即可。

## 五、开始捐款

```powershell
npm start
```

程序启动后会显示本次任务摘要并倒计时 5 秒（期间可按 `Ctrl+C` 取消）：

```
============================================================
⚠️  警告: 这是涉及真实金钱交易的自动化脚本！
============================================================
- 捐款金额: 0.01 元/次
- 捐款次数: 3 次
- 总金额: 0.03 元
============================================================

程序将在 5 秒后开始...
按 Ctrl+C 可以随时停止
```

### 自动执行流程

每一轮捐款的自动步骤如下：

1. 打开捐款项目页面
2. 点击「我要捐助」
3. 填入捐款金额
4. 勾选协议，点击「同意协议并捐款」
5. 在支付页面直接输入支付密码（6位）
6. 点击「确认付款」
7. 识别成功页面「您成功捐助 X 元，感谢您的爱心」
8. 等待 5 秒，开始下一轮
9. 达到设定次数后自动退出

## 六、注意事项

### 运行期间
- **不要手动操作浏览器**，程序正在控制浏览器，人工干预会导致流程失败
- 保持电脑不进入睡眠状态
- 保持网络畅通

### 中断与恢复
- 程序会在 `data/execution-log.json` 中记录已完成的次数
- 如果中途关闭程序，**重新运行 `npm start` 会从中断处继续**，不会重复执行已成功的轮次
- 如需从头开始，删除 `data/execution-log.json` 文件（或将其中 `completedCount` 改为 `0`）

### 登录状态失效
如果运行时提示未登录或被重定向到登录页，重新执行：
```powershell
npm run login
```

### 查看日志
- 实时日志：终端输出
- 历史日志：`data/donation.log`
- 执行记录：`data/execution-log.json`

## 七、常见问题

**Q：运行 `npm start` 报错"PAY_PASSWORD is required"**
A：`.env` 文件未创建或未填写 `PAY_PASSWORD`，参考第三节重新配置。

**Q：浏览器打开后停留在登录页面**
A：登录状态已失效，执行 `npm run login` 重新登录。

**Q：密码输入后提示"密码错误"**
A：检查 `.env` 中的 `PAY_PASSWORD` 是否正确（注意不要有空格）。

**Q：想捐更多次数怎么办？**
A：修改 `config/default.json` 中的 `totalTimes`，然后删除 `data/execution-log.json`，重新运行。

**Q：如何更换捐款项目？**
A：将捐款项目页面的 URL 复制到 `config/default.json` 的 `url` 字段。

## 八、文件结构速查

```
项目根目录/
├── .env                    ← 填写密码和银行卡信息（需自行创建）
├── .env.example            ← 配置模板，复制后改名为 .env
├── config/
│   └── default.json        ← 捐款次数、金额、等待时间
├── data/
│   ├── cookies.json        ← 登录状态（自动生成）
│   ├── execution-log.json  ← 执行记录（自动生成）
│   ├── donation.log        ← 运行日志（自动生成）
│   └── screenshots/        ← 错误时自动截图（自动生成）
└── src/                    ← 源代码（一般无需修改）
```
